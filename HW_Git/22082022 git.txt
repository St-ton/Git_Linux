Зачем нужен GIT и какие задачи решают с его помощью?
Это система контроля версий.
Позволяет фиксировать точки контроля при разработке на каком-либо языке. В любой момент можно обратиться к определенной точке сохранения

Что такое репозиторий GIT и где он находится?
Это так называемая база сохраненных контрольных точек процесса разработки.
Первоначально находится локально в папке .git. Но можно выгружать на облако, одним из которых является GitHub

Для чего используется файл .gitignore?
В этом файле указывается папки и файлы, которые исключаются из сохранения в репозиторий. Например логи или какие то кеш-файлы

Что такое "коммит"?
Коммит - это собсвенно само сохраненние файлов с измененниями в репозиторий.

Укажите базовые команды в рамках работы с GIT:
git init – создает новый репозиторий
git config --global -l – проверить настройки git
git status – проверка статуса репозитория
git add файл – поставить файл на учет в git и каждый раз этой командой его добавляем после каких-либо изменений
git commit – создаем коммит
git log – посмотреть логи, т.е. все коммиты
vim  .gitignore – создаем файл для игнорированных файлов/папок
git checkout id - откат на сохраненный коммит по его id


Настройка аккаунта локального пользователя.
git config --global user.name 'Stanislav Tonkich' 
git config --global user.email 'st.tonkikh@gmail.com' 
git config --global core.editor 'code' (или 'vim')
