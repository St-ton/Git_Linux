First level: Коротко опишите:
  • что такое ветка в GIT
  • Ветка - это ответвления от основного кода (как резервная копия наших файлов программного продукта), который создаем для работе сокдом. Создав ветку и работая в ней мы не беспокоимся за сохранность кода в основной ветке (мастера) от которой мы ответвлились. Ветки также позволяют нескольким разработчикам вести свои разработки паралеьно, т.е. они работают каждый как бы со своей веткой, в конце которые можно все их соединить в основную.
  • какое отношение имеют ssh-ключи к GITHUB
  • Прописав ssh-ключи на гитхабе мы работаем с ниб без логина/пароля пользователя. Это позволяет используя ключ на разных компах (домашний, рабочий и т.д.) связать их с моим репозитарием без логина/пароля
  • что позволяет делать сервис GitHub Pages
  • Это как временный хостинг. Он позвволяет посмотреть наш проект в браузере как аналог Live Server в VSC, только проект имеет свой адрес в интернет (как сайт) и его по ссылке может посмотерть любой пользователь
  • Укажите команды, которые выполняют след. действия (GIT):
  • создание, удаление веток;
  • git branch name_branch – создание новой ветки
    git branch -d name_branch
  • переключение между ветками;
  • git checkout name_branch
  • загрузка и удаление веток на GitHub.
  • git push origin name_branch
    git push origin –delete name_branch
